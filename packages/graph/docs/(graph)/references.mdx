## Normative References

- W3C DID Core: Fide IDs use DID method-style identifier syntax and resolution expectations.
- JSON-LD 1.1: Statement data and context compatibility assumptions rely on JSON-LD semantics.
- RDF 1.1 Concepts: FCP statement model aligns with graph assertion concepts and identifier-based nodes.
- Verifiable Credentials Data Model: Relevant for interoperable credential-like claims layered on top of FCP assertions.

## Informative References

- PROV-O: Useful for provenance and revision linkage patterns.
- OWL 2: Useful for strict equivalence and logical identity mapping.
- SKOS: Useful for taxonomy and concept hierarchy modeling.
- Schema.org: Useful for common predicate vocabulary and broad ecosystem interoperability.
